# 房地产户型盘点工具 — 使用说明书

---

## 一、工具简介

本工具用于将天朗房研网上的项目供应截图和成交截图，自动识别并提取表格数据，经过户型合并和指标计算，输出标准化的户型盘点 Excel 报告。

**核心能力**：截图 OCR 识别 → 数据合并 → 指标计算 → Excel 输出，全流程自动化。

---

## 二、工具工作流

```
第 1 步：准备 5 张截图
  ├── 整盘供应情况截图（1 张）
  ├── 整盘成交情况截图（1 张）
  └── 特定月份成交截图（3 张）

第 2 步：打开工具网页，填写项目信息
  ├── 项目名称（必填）
  ├── 板块（选填）
  ├── 开盘时间（选填）
  └── 容积率（选填）

第 3 步：拖拽上传 5 张截图，填写月份标签

第 4 步：点击「开始处理」
  ├── Qwen-VL 大模型识别截图中的表格
  ├── 自动合并户型顶/底变体
  ├── 自动计算户配、去化率、库存等指标
  └── 生成 Excel 文件

第 5 步：预览结果 → 下载 Excel → 手动补填少量字段
```

---

## 三、使用方法

### 3.1 启动工具

```bash
# 在项目目录下运行
python app.py
```

浏览器打开 **http://localhost:8080**。

### 3.2 截图准备

从**天朗房研网**截取以下 5 张图：

| 序号 | 截图类型 | 包含列 |
|------|---------|--------|
| 1 | 整盘供应情况 | 居室、面积范围、供应套数 |
| 2 | 整盘成交情况 | 居室、面积范围、成交套数、成交面积、成交均价 |
| 3-5 | 特定月份成交情况 | 同整盘成交（3 个不同月份） |

**截图要求**：PNG / JPG 格式，表格文字清晰可见。

### 3.3 填写信息并上传

1. 输入**项目名称**（会自动填入输出 Excel 的竞品列）
2. 可选填写板块、开盘时间、容积率
3. 将 5 张截图分别拖入对应上传区域
4. 为每个月度截图填写**月份标签**（如 `2024-05`）
5. 点击「开始处理」，等待约 10-30 秒

### 3.4 查看结果

处理完成后：
- 页面显示**数据预览表格**，黄色底色的列为需手动填写
- 点击「下载 Excel」获取正式报告

## 四、输出模板说明

生成的 Excel 包含以下列（按顺序）：

| 列名 | 来源 | 说明 |
|------|------|------|
| 板块 | **手动填写** | 项目所在板块 |
| 竞品 | 自动填入 | 项目名称 + 开盘时间 + 容积率 |
| 业态 | **手动填写** | 如 1T2-小高、1T2-洋房 |
| 编码 | 自动提取 | 从居室列提取户型编码，如 A户型 |
| 户型 | 自动映射 | 3室2厅1卫 → 三室一卫 |
| 户型面积 | 自动提取 | 来自供应截图的面积范围 |
| 建面分段 | 自动计算 | 根据面积中位数判断分段 |
| 整盘套数 | 自动计算 | = 供应套数 |
| 户配 | 自动计算 | = 该户型整盘套数 ÷ 项目整盘总套数 |
| 供应套数 | 自动提取 | 含顶/底变体合并 |
| 成交套数 | 自动提取 | 含顶/底变体合并 |
| 成交均价 | 自动提取 | 优先标准户型 |
| 整盘去化率 | 自动计算 | = 该户型成交套数 ÷ 该户型整盘套数 |
| 已供去化率 | 自动计算 | = 该户型成交套数 ÷ 该户型供应套数 |
| 已供去化占比 | 自动计算 | = 该户型成交套数 ÷ 整盘成交总套数 |
| 月份 1/2/3 | 自动提取 | 上=成交套数，下=成交均价 |
| 近 3 月月均销量 | 自动计算 | 三个月成交套数的平均值 |
| 近 3 月月均销量占比 | 自动计算 | = 该户型月均销量 ÷ 所有户型月均销量之和 |
| 已取证库存 | 自动计算 | = 供应套数 − 成交套数 |
| 整盘库存 | 自动计算 | = 整盘套数 − 成交套数 |

最后一行为**项目合计行**，编码、户型、户型面积合并为"合计"，指标自动汇总。

## 五、核心业务规则

### 5.1 户型顶/底变体合并

供应和成交截图中存在的 "A户型-底"、"A户型-顶" 会自动合并到 "A户型"：

- **供应套数** = A户型 + A户型-底 + A户型-顶
- **成交套数** = A户型 + A户型-底 + A户型-顶
- **成交均价 / 户型面积** = 以 A户型（标准户型）为准
- **成交面积** = 不聚合
- 如果没有标准户型，优先取 `-底1`，其次 `-底2`

### 5.2 室厅卫 → 户型映射

| 原始文字 | 映射结果 |
|---------|---------|
| 3室2厅1卫 | 三室一卫 |
| 4室2厅2卫 | 四室二卫 |
| 2室2厅1卫 | 二室一卫 |
| 复式 | 复式 |
| 无室厅卫 | 无室厅卫 |

> 规则：取"室"和"卫"前的数字转中文，省略"厅"的描述。

### 5.3 建面分段

根据户型面积中位数，自动归入以下分段：

```
(0,65] (65,75] (75,85] (85,95] (95,105] (105,115] (115,125]
(125,135] (135,150] (150,165] (165,180] (180,200] (200,220]
(220,240] (240,260] (260,+∞)
```

### 5.4 编码提取

从居室列文本中自动提取户型编码：

```
"A户型 3室2厅1卫"   → 编码 = "A户型"
"A户型-底 2室2厅1卫" → 编码 = "A户型"（去掉-底/顶后缀）
"C1户型-底1 3室2厅2卫" → 编码 = "C1户型"
```

## 六、注意事项

1. **截图质量**：确保截图清晰、表格完整，避免模糊、截断或遮挡
2. **API Key**：首次使用需在 `.env` 文件中配置通义千问 API Key（获取地址：https://dashscope.console.aliyun.com/apiKey）
3. **处理时长**：5 张截图通常 10-30 秒完成识别
4. **手动字段**：板块、业态需要你在 Excel 中手动补充
5. **数据校验**：建议下载 Excel 后快速核对关键数据（整盘供应套数、成交套数）

## 七、常见问题

**Q: 处理失败怎么办？**
A: 刷新页面重新上传，通常是因为某张截图识别异常。如果反复失败，检查 API Key 是否有效。

**Q: 生成的 Excel 中部分数据不对？**
A: 可能是截图清晰度问题导致 OCR 识别偏差。可以对比原截图，手动修正个别数据。

**Q: 月度成交截图少于 3 张？**
A: 至少需要 1 张月度截图，多余的月份可以不上传。

**Q: 如何更新 API Key？**
A: 编辑项目目录下的 `.env` 文件，修改 `DASHSCOPE_API_KEY=你的新Key`，重启工具即可。
